### Task 2: Add compression pipeline to ctx_execute_file

**Files:**
- Modify: `server/index.js:364-393` (ctx_execute_file handler, after intent-based search block)

- [ ] **Step 1: Add compression pipeline to ctx_execute_file**

In `server/index.js`, replace the raw stdout append (lines 390-393) with a try/catch-wrapped compression pipeline, matching the pattern from ctx_execute:

```javascript
    // ─── Compression pipeline ───
    let compressed = stdout;
    try {
      const toolPattern = `file:${language}`;
      const learner = getLearner();
      const sessionEvents = getRecentSessionEvents();
      const weights = learner.getWeights(toolPattern);
      const source = `file:${files[0]}:${Date.now()}`;

      // Index full output to KB before compression
      if (Buffer.byteLength(stdout, 'utf8') > INTENT_SEARCH_THRESHOLD) {
        try {
          const store = await getStoreAsync();
          store.index({ content: stdout, source });
          sessionStats.bytesIndexed += Buffer.byteLength(stdout, 'utf8');
        } catch {}
      }

      const compResult = compress(stdout, {
        toolName: toolPattern,
        command: code,
        sessionEvents,
        learnerWeights: weights,
        sourceLabel: source,
      });
      compressed = compResult.compressed;

      for (const decision of compResult.stats.decisions) {
        learner.recordDecision({
          toolPattern,
          contentHash: decision.contentHash,
          contentPreview: decision.contentPreview,
          sessionContext: sessionEvents.slice(0, 3).map(e => e.data).join('; '),
          sourceLabel: source,
        });
      }

      trackCompression(toolPattern, compResult.stats.originalBytes, compResult.stats.compressedBytes);
    } catch (compErr) {
      process.stderr.write(`[context-mode] execute_file compression error: ${compErr.message}\n`);
    }

    output += compressed;
```

This replaces the existing `output += stdout;` on line 390.

### Task 3: Fix ctx_fetch_and_index silent failure on network error

**Files:**
- Modify: `server/index.js:607-625` (ctx_fetch_and_index after subprocess completes)

- [ ] **Step 1: Add empty-stdout check before JSON.parse**

In `server/index.js`, after the `exitCode !== 0` check (line 607-609) and after `const stdout = result.stdout || '';` (line 611), add an empty-content guard before the JSON.parse block:

```javascript
    if (result.exitCode !== 0) {
      return makeErrorResponse(`Failed to fetch ${url}: ${result.stderr || 'unknown error'}`);
    }

    const stdout = result.stdout || '';
    const rawBytes = Buffer.byteLength(stdout, 'utf8');
    sessionStats.bytesSandboxed += rawBytes;

    // Guard: empty stdout means fetch produced no content
    if (!stdout.trim()) {
      return makeErrorResponse(
        `Fetch failed: no content returned from ${url}. Check the URL is accessible and try again.`
      );
    }

    // Parse JSON envelope from sandbox
```

This ensures we never reach `JSON.parse('')` and never index empty content.
