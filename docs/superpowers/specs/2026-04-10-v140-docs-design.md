## USER-MANUAL.md

### What to add

1. **Token Compression section** — Explain in layperson terms that context-mode automatically shrinks tool output so conversations stay sharp longer. Cover:
   - What happens automatically (no user action needed)
   - How to check savings: run `ctx_stats` and read the output
   - The error safety guarantee: errors/warnings are never compressed away
   - What "98% context reduction" actually means in practice

2. **Reading ctx_stats** — Walk through the new output format:
   - Token savings (compressed vs. sandboxed)
   - Cost estimates (Opus/Sonnet/Haiku)
   - Compression breakdown by tool
   - Learner retrieval rate and confidence
   - Lifetime stats

3. **Updated routing section** — 23 matchers now (up from 14). Explain which commands get auto-redirected: git log, git diff, npm test, pytest, cargo build, docker build, make, npm install, pip install. Explain pass-through conditions in plain language ("if you pipe output through grep or limit results, your command runs normally").

4. **New error messages** — ctx_fetch_and_index now returns explicit errors on failed fetches. Add to troubleshooting: "Fetch failed: no content returned" means the URL was unreachable or returned empty.

5. **"How It Works" appendix** — For technically curious users:
   - 3-stage pipeline: deterministic stripping → pattern-based compression → session-aware relevance
   - 10 pattern matchers and what each does
   - Error safety invariant: error keywords + 2-line context protection
   - Self-learning feedback loop: compression decisions → retrieval signals → retention weight adjustment
   - 2KB threshold: small outputs only get ANSI stripping

6. **Glossary updates** — Add: compression, retention score, retrieval rate, token, context window, pipeline

### Tone

Main sections: warm, patient, zero jargon without definition. "How It Works" appendix: still accessible but can use technical terms that were defined in the glossary.
