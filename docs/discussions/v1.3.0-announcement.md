# context-mode v1.3.0 — Self-Learning Token Compression Engine

**Category:** Announcements (pin this post)

---

We're excited to release **context-mode v1.3.0**, the biggest feature update since launch. This release adds a self-learning compression engine that intelligently reduces the tokens returned to your context window — while never dropping errors, failures, or content relevant to your current work.

## What's New

### Token Compression (3-Stage Pipeline)

Every tool output that passes through `ctx_execute` or `ctx_batch_execute` now goes through a compression pipeline before entering your context:

1. **Deterministic stripping** — ANSI escape codes, terminal cursor sequences, carriage return overwrites, BOM markers, trailing whitespace, and duplicate blank lines are removed losslessly.

2. **Pattern-based compression** — 10 tool-specific matchers understand the structure of common outputs:
   - **jest/vitest**: Passing tests collapse to a count. Failures preserved verbatim with full stack traces.
   - **pytest**: Same — passes summarized, failures kept.
   - **git log**: Deduplicates merge commits, caps at 30 entries, preserves commits mentioning your session-relevant files.
   - **git diff**: Preserves hunks for files you're working on, summarizes the rest.
   - **cargo build**: Strips "Compiling crate v1.0.0" lines, keeps warnings/errors/summary.
   - **npm/pip install**: Strips download progress, keeps summary + warnings.
   - **docker build**: Strips cache hits, keeps build steps and image ID.
   - **make/cmake**: Strips gcc/g++ invocations, keeps warnings/errors.
   - **Directory listings**: Collapses node_modules/.git/__pycache__ contents.

3. **Session-aware relevance** — Scores each content block against your current session (files you've edited, patterns you've been working on). Low-relevance blocks are summarized. High-relevance content is preserved.

### Self-Learning Feedback Loop

The compression engine gets smarter over time. When Claude uses `ctx_search` to retrieve previously compressed content, the learner detects this and adjusts retention weights for that tool pattern. Tools whose compressed output is frequently retrieved get higher retention (less aggressive compression). Tools that are never retrieved get compressed more aggressively.

Run `ctx_stats` to see:
- Learner accuracy (retrieval miss rate)
- Lifetime token savings across sessions
- Estimated cost savings for Opus, Sonnet, and Haiku pricing

### 9 New Routing Matchers

PreToolUse now intercepts 23 Bash command patterns (up from 14). New matchers:
- `git log` / `git diff` (with smart pass-through for `--oneline`, `-n`, `--stat`, single-file diffs)
- `npm test` / `npx jest` / `npx vitest`
- `pytest` / `python -m pytest`
- `npm install` / `npm ci`
- `pip install`
- `cargo build` / `cargo test`
- `docker build`
- `make` / `cmake --build`

All matchers have pass-through conditions — piped commands (`| grep`, `| head`) and bounded queries pass through unchanged.

### Rewritten ctx_stats

`ctx_stats` now shows:
- Token savings with compression breakdown by tool pattern
- Estimated cost savings at current Anthropic pricing (Opus $15/M, Sonnet $3/M, Haiku $0.80/M)
- Learner accuracy and confidence metrics
- Lifetime aggregate statistics across sessions

## Error Safety Guarantee

The compression engine has a hard invariant: **lines containing error keywords are never compressed.** This includes Error, TypeError, FAIL, warning, WARN, panic, exception, and Traceback. These lines plus 2 lines of context above and below are always preserved verbatim, regardless of compression stage or relevance score.

## How to Update

```bash
npx --yes --package=github:scottconverse/context-mode context-mode
```

Or in Claude Code:
```
/plugin marketplace add scottconverse/context-mode
```

Start a new session after updating.
